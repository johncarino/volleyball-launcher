# Volleyball Launcher — PWM Motor Controller

Controls two DC hoverboard motors via PWM to launch a volleyball. Runs on a **BeagleY-AI**.

## Hardware

- **BeagleY-AI** (TI AM67A)

- **2x DC hoverboard motors** (36V)
- **Power supply:** 36V DC

### Wiring

| Motor | GPIO | Header Pin | PWM Channel | Driver Board Pins |
|-------|------|------------|-------------|-------------------|
| Motor 1 | GPIO12 | Pin 32 | pwmchip3/pwm1 (EPWM0 ch B) | P (signal), G (ground) |
| Motor 2 | GPIO15 | Pin 10 | pwmchip3/pwm0 (EPWM0 ch A) | P (signal), G (ground) |

**Driver board setup:**
- PWM short-circuit pads soldered (enables external PWM input)
- On-board speed potentiometer turned to minimum
- Only P and G pins connected (no DIR, BRAKE, or STOP)

## Project Structure

- `hal/`: Hardware abstraction layer — PWM control via Linux sysfs
- `app/`: Application code — motor control logic and `main()`
- `build/`: Generated by CMake; stores all temporary build files (may be deleted to clean)

## BeagleY-AI Setup

### Enable PWM Overlay

Edit the boot config on the BeagleY-AI:

```bash
sudo nano /boot/firmware/extlinux/extlinux.conf
```

Add this line under the `label microSD (default)` section:

```
fdtoverlays /overlays/k3-am67a-beagley-ai-pwm-epwm0-gpio12.dtbo
```

Reboot for the overlay to take effect.

### Verify PWM

```bash
# Check pwmchip exists
ls /sys/class/pwm/pwmchip3/

# Confirm 2 channels available
cat /sys/class/pwm/pwmchip3/npwm    # should output: 2

# Verify GPIO12 mapping
realpath /dev/hat/pwm/GPIO12
```

## Building

### Prerequisites

- CMake (`sudo apt install cmake`)
- ARM64 cross-compiler (`sudo apt install gcc-aarch64-linux-gnu`)
- VS Code with **CMake Tools** extension

### Cross-Compile for BeagleY-AI

1. Delete the `build/` folder: `rm -rf build`
2. In VS Code: **Ctrl+Shift+P → CMake: Select a Kit** → choose `aarch64-linux-gnu-gcc`
3. **Ctrl+Shift+P → CMake: Configure**
4. Build with **Ctrl+Shift+B** or **Ctrl+Shift+P → CMake: Build**

The executable is automatically copied to `/home/john/Desktop/BeagleY-AI/public/motor_control`.

### Manual Build (command line)

```bash
rm -rf build/
cmake -S . -B build
cmake --build build
```

## Running on the BeagleY-AI

Copy the executable to the board and run with `sudo` (required for sysfs PWM access):

```bash
sudo ./motor_control
```

Press **Ctrl+C** to stop the motors cleanly.

## API Overview

The PWM HAL provides:

| Function | Description |
|----------|-------------|
| `pwm_init()` | Export both PWM channels and configure defaults |
| `pwm_set_frequency(hz)` | Set frequency for both motors (50–20,000 Hz) |
| `pwm_set_duty_cycle(motor, %)` | Set speed per motor (0–100%) |
| `pwm_enable(motor, on/off)` | Enable/disable a motor |
| `pwm_cleanup()` | Disable both motors and unexport channels |

Use `PWM_MOTOR_1` and `PWM_MOTOR_2` constants to address each motor.

## Finer Points

- HAL headers are included as `#include "hal/pwm.h"` to distinguish low-level access from application code.
- Re-run CMake configure when adding or removing `.h`/`.c` files (or resave `CMakeLists.txt` to trigger it in VS Code).
- Both motors share the same PWM frequency (same EPWM0 timer) but have independent duty cycles.
